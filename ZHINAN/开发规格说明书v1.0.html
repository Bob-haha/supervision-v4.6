全系统页面开发规格书（终版）

本文档整合所有前端页面的详细开发规格，并应用了全面对照分析后的修正与调整。内容包括：首页看板、新建任务、任务详情、我的任务列表、督办看板，以及流程配置、明细模板配置、字段管理、态势配置、文件管理、类型与标签配置等配置管理页面。每个页面的数据、交互、状态管理和权限边界均已明确，可直接指导开发。

1. 首页看板（ /home ）

1.1 页面概述

首页看板是系统的信息中枢与个人工作入口，包含三个区域：

区域 可见范围 核心内容
关区业务态势 关领导、督办员（职位包含"关长/副关长"或 user_roles.role=inspector） 关级指标卡片、多标签页任务列表、领导批示置顶
科级业务态势 当前用户所在科室所有人员（departmentId 相同） 本科室指标卡片、多标签页任务列表、科长关注置顶
个人工作台 所有用户 待办、关注、我发起的、送阅、提取任务五个标签页

所有数据从本地 IndexedDB 获取，前端聚合计算呈现，不产生后端请求。P2P 同步只负责将授权任务推送到本地。

1.3 关区业务态势（ CustomsDashboard 组件）

1.3.1 显示权限

  • 显示条件（任一满足即显示整个 #customs-card）：
    • 当前用户 position 字段包含"关长"或"副关长"；
    • 当前用户在 user_roles 表中有记录且 role = 'inspector'。
  • 权限判断在 Vue 组件 computed 中根据 Pinia 用户 store 计算。

1.3.2 指标卡片区

数据准备

  • 从 metric_definitions 表筛选 level = 'customs_level' 且 visibility 允许当前用户查看的指标。
    • visibility 取值：'all' 所有人可见；'role:leader' 仅关领导；'role:inspector' 仅督办员。关领导可查看 inspector 指标（按业务逻辑配置）。
  • 当前用户自定义指标从 user_dashboard_config 的 customsMetrics 数组获取，若该数组为空或未配置，则使用系统推荐指标（isRecommended = true）。
  • 最终展示的指标 = 自定义列表中存在的指标ID ∩ 用户有权限访问的指标。

聚合计算

每个指标卡片的值由以下步骤实时计算：

  1. 读取指标定义的 config 对象。
  2. 根据 config.mode 区分数据筛选方式：
    • 按条件筛选（mode='filter'）：使用 taskTypeIds、filterTags、timeRange 条件从 tasks 表筛选。
    • 指定具体任务（mode='specific'）：通过 specificTaskIds 直接从 tasks 表获取指定任务。
  3. 排除已归档任务（status != 'archived'）。
  4. 对筛选出的任务集合，根据 config.fieldId 和 config.aggFunc 聚合：
    • fieldId = '__task_count__' 时统计任务总数（COUNT）。
    • 否则遍历任务的 dataSheetRows，取出指定字段值（忽略不存在的字段），应用聚合函数（SUM、AVG、MAX、MIN、COUNT）。
  5. 计算环比变化（可选）：对比上一周期值。

卡片渲染

  • 使用 MetricCards 组件，传入指标数组。
  • 每个卡片结构：
<div class="data-card" @click="openDrillDown(metric)">
    <div class="label">{{ metric.name }}</div>
    <div class="value">{{ formattedValue }}</div>
    <div class="sub">{{ changeDescription }}</div>
</div>
  • 数值自动格式化（万/亿或千分位）。
  • 鼠标悬停显示完整数字 tooltip。

点击穿透

  • 点击卡片数值区域，打开 DrillDownModal 组件。
  • 模态框标题：指标名称 - 构成明细
  • 内容：使用 TaskList 组件（只读模式）展示所有聚合到的任务，列固定为任务标题、部门、当前环节、时限、预警标记。
  • 点击某行跳转到任务详情页 /task/:id。
  • 模态框底部显示任务总数。

个性化配置

  • 指标卡片区右上角按钮"自定义指标"。
  • 点击打开 MetricConfigModal，接收 level="customs"：
    • 左侧列出当前用户有权查看的关级指标（带复选框）。
    • 右侧为已选列表（可拖拽排序）。
    • 保存时更新 user_dashboard_config 的 customsMetrics 数组（只存指标ID），并触发 P2P 同步配置。
    • 关闭后立即刷新卡片区。

自动刷新

  • onMounted 时启动定时器，每 30 秒调用一次聚合更新。
  • 监听 P2P 数据变更事件（p2p-data-updated），若涉及 tasks 表则立即重新聚合。
  • 使用 watchEffect 跟踪用户配置变化。
  • 组件卸载时清除定时器和事件监听。

1.3.3 标签页系统

标签页数据模型

存储于 user_dashboard_config.tabs 数组中（type = 'customs_task_list'）。结构：

{
                "id": "UUID",
                "label": "标签名称",
                "type": "customs_task_list",
                "config": {
                "columns": ["title", "taskType", "tags", "department", "deadline", "progress"],
                "filters": {
                "taskTypeIds": [],
                "statuses": [],
                "tagIds": [],
                "departmentIds": [],
                "keyword": "",
                "timeRange": { "start": null, "end": null }
                },
                "sortBy": { "field": "deadline", "order": "asc" },
                "isDefault": false
                }
            }

系统预置标签页（不可删除）：

  1. 全部任务：无筛选，显示所有有权任务（关闭按钮隐藏）。
  2. 年度重点任务：筛选 taskTypeIds 包含"年度重点任务"类型ID。
  3. 领导关注：筛选 priority = 'leadership_attention'。
  4. 逾期/临期：筛选逾期或临期任务（截止日期已过或7天内到期）。

交互

  • 新建：点击 + 按钮 → 命名 → 插入到 tabs 数组，设为激活，保存配置。
  • 切换：点击标签更新 activeTabId，重新渲染任务列表。
  • 重命名：悬浮编辑按钮，内联编辑修改 label。
  • 关闭：删除标签对象，若关闭当前激活标签则激活相邻；若只剩一个且不可关闭，则不做操作。
  • 所有操作后持久化 user_dashboard_config。

列选配置

  • "列选"按钮打开 ColumnSelectModal，可选列池从系统预定义字段中读取，用户勾选后更新当前激活标签的 config.columns。

高级筛选

  • "高级配置"按钮打开 AdvancedFilterModal，提供任务类型、状态、标签、部门、时间范围、关键词等筛选，以及排序规则。
  • 底部实时预览符合条件任务数和逾期任务数。
  • 应用后将条件写入当前标签的 config.filters 和 config.sortBy，保存配置。

任务列表渲染

  • 使用 TaskList 组件，接收过滤条件、排序列、可见列等 props。
  • 从 tasks 表查询任务，应用过滤、排序，动态展示列。
  • 每行显示预警灯、标签、各列数据。行点击跳转详情。
  • 右侧操作按钮（仅当用户有权限时显示）：
    • 批示（关领导可见）：弹出输入框，提交后调用 addLeaderComment(taskId, content) 方法，向任务的 activityLog 数组追加一条 type='leader_comment' 的记录，并记录批示人和时间。
    • 催办（督办员可见）：打开催办弹窗，可输入催促留言，向任务动态区插入一条 type='urge' 的记录。
    • 详情：跳转。

1.3.4 领导批示置顶区

  • 组件 LeaderCommentBar，嵌入标签页栏下方、任务列表上方。
  • 数据源（调整后）：查询 tasks 表，筛选当前用户可见任务，检查 activityLog 数组中是否存在 type='leader_comment' 且 timestamp 在近 7 天内的记录。按时间倒序排列。
  • 每条显示：
    • 任务标题（可点击跳转）
    • 当前环节
    • 最新批示摘要（截取前 50 字）
    • 批示人和时间
  • 若无批示，隐藏该区域。
  • 点击跳转任务详情并定位到批示动态。

1.4 科级业务态势（ SectionDashboard 组件）

1.4.1 显示权限

  • 所有用户，只要所属部门有效（personnel.departmentId 非空）即显示。
  • 数据范围：所有聚合和任务列表均限制为本部门（通过 departmentId 过滤）。

1.4.2 指标卡片区

  • 从 metric_definitions 读取 level = 'section_level' 的指标定义，并校验 visibility（若配置为仅科长可见，则检查当前用户是否为本科室负责人）。
  • 自定义指标：从 user_dashboard_config.sectionMetrics 数组读取。点击"自定义指标"打开 MetricConfigModal，传入 level="section"。
  • 聚合计算：筛选 tasks 时，增加条件：任务的责任矩阵中 primary 或 cooperative 任一 departmentId 等于当前用户所属部门ID。
  • 其他交互同关区态势。

1.4.3 标签页系统

  • 独立存储，type = 'section_task_list'。
  • 默认标签页：全部、科长关注、临期/逾期。
  • 用户可自定义，逻辑一致。

1.4.4 任务列表

  • 自动过滤本科室任务（主办或协办部门为本部门）。
  • 列表显示主办/协办标识（增加"角色"列），显示当前环节、预警状态。

1.4.5 科长关注置顶区

  • 组件 SectionFocusBar。
  • 数据源：从 tasks 表中筛选 starredBy 包含本部门 isLeader=true 的人员ID的任务，且状态未完成。
  • 若非科长，只读查看；科长可在此快速添加/取消关注（点击星标）。
  • 若无关注任务，隐藏。

1.5 个人工作台（ PersonalWorkbench 组件）

1.5.1 整体设计

  • 五个标签页使用自定义标签组件。每个标签页对应一个子组件。

1.5.2 待办任务标签页

  • 查询条件：responsibleMatrix 中 personnelId 等于当前 peerId，且状态为 pending 或 in_progress。
  • 列表每行显示：任务标题、来源标签、当前环节名称、时限 + 预警标记、最新动态摘要。
  • 若用户在同一任务中既有主办又有协办，显示为一条记录，角色显示"主办/协办"。
  • 点击行跳转详情。支持复选框和批量送阅。

1.5.3 关注任务标签页

  • 筛选 starredBy 包含当前 peerId。置顶显示 priority = 'leadership_attention' 的任务。支持批量送阅。

1.5.4 我发起的任务

  • 筛选 createdBy === currentPeerId。按状态分组展示（进行中、已完成）。批量送阅可用。

1.5.5 送阅任务

  • 筛选 sharedWith 包含当前 peerId。仅提供"详情"按钮，不可批量操作。

1.5.6 我的提取任务

  • 筛选 extractionConfig 不为空且 extractionConfig.executorId === currentPeerId。
  • 列表上方显示批量操作栏："一键登录全部"、"批量手动提取"。
  • 每个任务列表项显示：标题、目标URL、状态指示灯、最近提取时间、下次提取时间。
  • 操作按钮：一键登录、手动提取、启用/禁用自动提取。
  • 提取结果更新通过 P2P 同步回本地，动态区自动添加记录。

1.5.7 批量送阅

  • 在待办、关注、我发起等标签页的任务列表中，每行有复选框。
  • 顶部工具栏显示"批量送阅"按钮，显示已选数量。
  • 点击打开 PersonSelector 模态框，支持多选人员。
  • 确认后，遍历勾选任务，将选中人员的 peerId 添加到 sharedWith 数组（去重），保存任务。
  • 提示"已成功送阅给 N 人"，触发 P2P 同步。

1.6 数据监听与刷新策略

事件触发时机影响范围实现
P2P数据同步远程任务数据写入本地 IndexedDB所有区域的指标卡片和任务列表全局事件 p2p-data-updated，监听 tasks 表变更，触发对应组件刷新
用户配置变更自定义指标、标签页修改保存当前页面相应区域直接调用刷新方法
扩展提取结果更新扩展通过 P2P 更新了 tasks 表提取任务列表、关联任务动态同上
定时器每30秒指标卡片数值重算（不重新请求数据）setInterval

优化：

  • 聚合计算使用 Dexie 索引加速（taskTypeId, tags, departmentId, deadline）。
  • 任务列表若数据量大（>200），使用虚拟滚动。
  • 指标卡片计算采用缓存，比较 tasks 表的 updatedAt 最大值避免重复计算。

1.7 异常处理与边界

  • 指标计算错误：显示"--"，控制台输出详细错误。
  • 配置丢失：自动使用默认配置（预置标签页，显示推荐指标）。
  • 扩展通信失败：提示"浏览器扩展未启动或未连接"。
  • 权限数据缺失：按最低权限处理。
  • 标签页关闭最后一个：按钮失效，提示"至少保留一个标签页"。

2. 新建任务页（ /task/new ）

2.1 页面概述

所有用户均可访问。通过表单填写基本信息、指派责任人、挂载流程和明细模板、预分配子任务，创建后写入 IndexedDB，并通过 P2P 同步给所有 authorizedPeers。

2.2 组件拆分

组件文件说明
views/NewTask.vue页面容器管理整个创建流程
components/task/TaskBasicInfo.vue表单区块：标题、描述、时限、优先级、来源
components/task/CascadeSelect.vue级联选择器（通用：一级→二级）用于任务类型和标签
components/task/PersonnelDeptSelector.vue主办/协办部门人员选择器树形部门→人员多选
components/task/TemplatePreview.vue流程步骤和明细表预览
components/task/SubtaskPreAllocator.vue子任务预分配列表（动态添加行）核心复用组件
components/task/ExtractionConfigInline.vue外部提取配置区块可折叠

2.3 功能点

2.3.1 基础信息录入

字段tasks 字段控件必填说明
任务标题title<input>是最大长度 100
任务描述description<textarea>否V1 纯文本
截止日期deadline<input type="date">否
优先级priority单选按钮组是normal/urgent/leadership_attention，默认 normal
任务来源source动态否结构 {type, reference}，手动创建 type 默认 manual

2.3.2 任务类型级联选择

  • 数据源：task_type_groups（一级）→ task_types（二级）。
  • 两个并排 <select>，选择二级类型后自动获取默认流程模板和明细模板ID，并在预览区域显示默认步骤/表头。
  • 若无默认模板，预览区提示"该类型无默认流程/明细模板"，用户仍可手动选择。

2.3.3 标签多选

  • 数据源：tag_groups → tags。
  • 使用级联选择，第二个下拉改为多选，已选标签以标签形式展示，可逐个删除。存储为 tags 数组（标签ID）。

2.3.4 主办/协办部门人员指派

  • 使用 PersonnelDeptSelector 树形选择。先选部门，再选人员（可缺省人员）。主办至少选择一个部门。
  • 存储为 responsibleMatrix；提交时自动计算 authorizedPeers。

2.3.5~2.3.9 其余功能点

包括流程模板选择、明细模板选择、子任务预分配（使用 SubtaskPreAllocator 组件，折叠区域默认收起，支持动态添加/删除行，可继承父任务流程/明细）、任务来源标记、外部数据提取配置等，详见完整规格书。

2.4 提交流程

  1. 验证：标题不为空，主办至少一个部门。
  2. 构建任务对象，生成 UUID。
  3. 写入 tasks 表。
  4. 循环创建子任务（若有），建立 task_relations，更新父子任务关系，合并 authorizedPeers。
  5. 触发 P2P 同步。提示"任务创建成功"，跳转 /task/:id。

2.5 状态与异常

  • 加载状态：全局遮罩，待类型、标签、部门、人员数据加载完成后显示表单。
  • 下拉数据加载失败：对应下拉置灰，显示"加载失败"并提供重试。
  • 提交中：按钮 loading，防止重复提交。
  • 提交失败：提示"任务创建失败，请重试"，记录错误日志。

3.3 区域一：任务头栏

  • 标题编辑：仅 createdBy 可编辑，点击 <h2> 变为 <input>，失焦或回车保存。
  • 类型标签：显示二级类型名称，点击可查看默认流程和明细配置（只读模态）。
  • 状态显示：映射 status 为中文（待处理/进行中/已完成/已归档），带颜色标签。
  • 时限与预警：根据 deadline 计算逾/临期/正常，显示红/黄/绿标签和指示灯。
  • 关注星标：starredBy 数组包含当前 peerId 则高亮，点击切换。
  • 送阅：按钮 → PersonSelector 多选 → 合并到 sharedWith 并保存。
  • 面包屑：若有父任务，显示"← 所属父任务：{标题}"，点击跳转。
  • 归档按钮（新增）：仅 createdBy 且 status === 'completed' 时显示，点击二次确认后 status='archived'，记录动态。

3.4 区域二：责任矩阵

  • 以卡片列表展示主办/协办信息，显示部门、人员（或"待分派"）、角色标签、反馈状态（pending/submitted/confirmed 对应灰/黄/绿）。
  • 逐级分派：仅当前用户为该部门负责人且条目无人员时，显示"添加经办人"，弹出 PersonSelector，更新 responsibleMatrix 并扩展 authorizedPeers，自动生成动态记录。
  • 分派反馈：责任条目本人可点击"反馈"，提交文本+附件，更新 feedbackStatus 为 submitted，插入 type='responsibility_feedback' 动态。

3.5 区域三：环节流程

  • 环节导航条：基于 stagesSnapshot 渲染步骤条，currentStageIndex 之前绿色打勾，当前高亮。
  • 选项卡按钮：根据当前环节 options 数组生成操作按钮。点击按钮触发 progressStage(nextStageIndex, optionLabel)：
    • 状态自动流转（新增逻辑）：若推进前 status === 'pending'，自动更新为 'in_progress'。若 nextStageIndex >= stagesSnapshot.length，将 status 设为 'completed'，记录完成时间。
    • 在动态区插入流转记录（type='stage_progress'）。
  • 环节约束：若 requireSupervisorConfirm 为 true，推进时生成待确认记录，上级确认后才真正推进。
  • 资源引用：模板文件下载、制度文件查看（内嵌 PDF 预览，跳转条款页码）、外部链接跳转。
  • 附件上传：上传后写入 files 表，动态区生成附件记录。
  • 流程配置入口：任务负责人可见，可选取模板替换当前快照或手动编辑环节，可"保存为常设流程"。

3.6 区域四：子任务区域

  • 子任务列表：通过 task_relations 表查询子任务，显示标题、负责人、状态、时限预警、进度。
  • 创建子任务：复用 SubtaskPreAllocator 组件（模态框），支持动态添加子任务草稿，可继承父任务流程/明细。确认后创建并建立关系。
  • 关联已有任务：搜索选择已有任务，选择汇总模式，创建 task_relations 记录。
  • 子任务进度自动汇总：监听子任务状态变化，在父任务动态区插入汇总记录。
  • 子任务相互可见：通过 authorizedPeers 合并实现。

3.7 区域五：动态与批示区

  • 数据模型：activityLog 数组，每条含 id、type、content、timestamp、actorPeerId、metadata。
  • 时间线渲染：按时间倒序，不同类型不同图标和样式。
  • 进展反馈输入：底部固定区域，文本+附件上传，提交后追加 type='feedback' 记录。
  • 领导批示：关领导可见专用输入框，提交后追加 type='leader_comment' 记录。不再使用 hasLeaderComment 字段，首页批示置顶区直接扫描 activityLog。
  • 负责人确认反馈：当有需要确认的条目时，显示"确认"按钮，更新 confirmedFeedback 并插入系统记录。

3.8 区域六：数据明细区

  • 根据 dataSheetFields 快照构建表格，从 system_fields 获取字段信息。
  • 新增行：末尾添加可编辑行，提交后生成 rowId，记录 creatorId。
  • 编辑行：仅创建人可编辑。删除行：仅创建人可删，二次确认。
  • 父子同构汇总：若 childAggregationMode === 'homogeneous'，自动汇总所有同构子任务数据行。
  • 异构汇总：根据 aggregationRules 渲染汇总卡片，点击可穿透查看子任务明细。
  • 快照查看：展示只读数据卡片，标注来源任务和时间。

3.9 提取配置面板

模态框：执行人、URL、映射配置、提取模式。保存时更新 tasks.extractionConfig，P2P 同步。

3.10 数据持久化与权限

  • 所有修改通过 saveTask() 调用 Dexie 的 put 操作。
  • 访问权限：若 authorizedPeers 和 sharedWith 均不包含当前用户，重定向首页并提示无权访问。
  • 操作权限：编辑标题（创建人）、分派（部门负责人）、反馈（责任人）、流程自定义（任务负责人）、删除任务（创建人，可在头栏提供删除按钮）。
